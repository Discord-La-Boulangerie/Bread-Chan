Commandes de Bread Chan

Découvrez les commandes puissantes de Bread Chan, le bot de modération pour plusieurs serveurs Discord. Que vous ayez besoin de gérer les membres, de personnaliser les interactions ou d'obtenir des informations, notre bot est là pour vous simplifier la tâche. Explorez les commandes disponibles pour améliorer la gestion de votre serveur Discord.

/ping

  • Description :simple, efficace, elle permet simplement d'indiquer la latence du bot.
  • Permissions requises :Aucunes.
  • Paramètres :
    • Aucun

bot_info

  • Description :Cette commande permet d'afficher les informations du bot, à savoir :
    • son nom
    • sa photo de profil actuelle
    • ses librairies, ainsi que leurs versions
  • Permissions requises : Aucunes.
  • Paramètres :
    • Aucun

genshin_info

  • Description :Cette commande permet d'afficher le profil Genshin Impact d'un joueur.
  • Permissions requises : Aucunes.
  • Paramètres :
    • `UID` : l'indentifiant du joueur que vous souhaitez rechercher.

ban_appeal

  • Description :Cette commande gère les demandes de débannissement. Elle vérifie si un utilisateur est banni sur un serveur Discord, puis affiche la raison du bannissement et offre la possibilité de soumettre une demande de débannissement.
  • Permissions requises : Aucunes, il faut en revanche être banni du serveur afin de pouvoir effectuer une demande.
  • Paramètres :
    • Aucun

rps

  • Description : Cette commande permet de jouer à shi-fu-mi avec Bread Chan (elle est très forte).
  • Permissions requises : Aucunes.
  • Paramètres :
    • `choix` : un choix entre `Rock`, `Paper` et `Scissors`.

/mute

  • Description : Cette commande permet de mute un utilisateur spécifié du serveur Discord.
  • Permissions requises : L'utilisateur qui exécute la commande doit avoir la permission de gérer des membres sur le serveur.
  • Paramètres :
    • `membre` : L'utilisateur que vous souhaitez mute.
    • `durée` : la durée du mute
    • `raison` (optionnel) : La raison du mute.
    • `fichier` (optionnel) : preuve potentielle attestant de la véracité du mute

kick

  • Description : Cette commande permet de kicker un utilisateur du serveur Discord.
  • Permissions requises : L'utilisateur qui exécute la commande doit avoir la permission de kicker des membres sur le serveur.
  • Paramètres :
    • `member` : L'utilisateur que vous souhaitez kicker.
    • `reason` (optionnel) : La raison du kick.

webhook

  • Description : Cette commande permet d'envoyer un message via un webhook dans un salon spécifié.
  • Permissions requises : L'utilisateur qui exécute la commande doit avoir la permission de gérer le serveur.
  • Paramètres :
    • `texte` : Le texte que le webhook doit envoyer.
    • `nom` : Le nom du webhook.
    • `channel` (optionnel) : Le salon où le message du webhook sera envoyé.
    • `file` (optionnel) : Une image qui peut servir de photo de profil pour le webhook.
